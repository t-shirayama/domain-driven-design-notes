# 整合性境界

Aggregate は整合性境界です。境界内では、同時に守る必要がある不変条件を保ちます。境界外の整合性は、結果整合性で扱うことがあります。

| 判断 | Aggregate に含める |
| --- | --- |
| 同時に必ず守る必要がある | 含める |
| 少し遅れて反映されてもよい | 分ける |
| 別チームや別文脈の関心 | 分ける |
| 参照だけでよい | ID 参照を検討する |

Aggregate を大きくすると、整合性は守りやすくなりますが、変更、保存、ロックの単位が重くなります。

たとえば注文では、「確定済みの注文に明細を追加できない」「合計金額は明細から計算する」のように、同時に守る必要があるルールを境界内に置きます。一方で、在庫引当や請求作成は別 Aggregate や別システムの関心になりやすく、イベントでつなぐ候補になります。

迷ったら、同じトランザクションで失敗させないと業務上困るかを確認します。困らないなら、境界を分けた方がモデルを小さく保てます。

**Aggregate の境界は、オブジェクトの親子関係ではなく不変条件から決めます。**
