# Entity と Value Object の判断

Entity と Value Object の判断では、「あとから同じものとして追跡する必要があるか」を見ます。追跡するなら Entity、値が同じなら同じものとして扱えるなら Value Object にします。

| 問い | Entity 寄り | Value Object 寄り |
| --- | --- | --- |
| ID で追跡するか | する | しない |
| 状態が変わっても同じものか | 同じ | 値が変われば別物 |
| 例 | 注文、顧客、契約 | 金額、期間、住所 |

住所は多くの場合 Value Object ですが、配送先住所を履歴として追跡する必要があるなら Entity として扱う場合もあります。

判断はシステム全体で固定ではなく、境界づけられたコンテキストごとに変わります。配送コンテキストでは住所を配送先として追跡し、請求コンテキストでは請求書に印字する値として扱うことがあります。

迷ったら、値が変わったときに「同じものが更新された」と言いたいのか、「別の値に差し替わった」と言いたいのかを確認します。

**型の種類は名前ではなく、その文脈での扱い方で決まります。**
