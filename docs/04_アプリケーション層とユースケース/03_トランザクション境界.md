# トランザクション境界

トランザクション境界は、どこまでを一度に成功または失敗させるかを決める境界です。DDD では、Aggregate の一貫性とユースケースの単位を見ながら決めます。

1 つのユースケースで複数の Aggregate を更新したくなる場合は、本当に同時整合性が必要かを確認します。必要でなければ、ドメインイベントや非同期処理で結果整合性に寄せます。

| 更新対象 | 考え方 |
| --- | --- |
| 1 つの Aggregate | 同一トランザクションで保存しやすい |
| 複数の Aggregate | 同時整合性が必要か確認する |
| 外部サービス | DB トランザクションに含めない |

外部 API 呼び出し、メール送信、メッセージ発行は、DB 更新と同じトランザクションで扱えないことが多いです。

**トランザクション境界は、技術都合だけでなく一貫性の要求から決める**必要があります。
