# 境界づけられたコンテキスト

境界づけられたコンテキストは、モデルと言葉が一貫した意味を持つ範囲です。同じ言葉でも、販売、請求、配送では意味が変わることがあります。

たとえば `Customer` は、販売では購入主体、請求では支払先、サポートでは問い合わせ元かもしれません。無理に 1 つの巨大な `Customer` にまとめると、変更理由が混ざります。

| コンテキスト | `Customer` の関心 |
| --- | --- |
| 販売 | 注文できるか |
| 請求 | 請求先と支払条件 |
| サポート | 問い合わせ履歴 |

境界を決めるときは、DB テーブルやチーム構成だけでなく、言葉の意味と変更理由を見ます。

**同じ名前でも意味が違うなら、別のモデルとして扱う**ことを検討します。
