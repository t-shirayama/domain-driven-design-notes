# ORM との距離

Entity Framework Core のような ORM は便利ですが、ドメインモデルが ORM の都合に引きずられることがあります。設計では、どの程度 ORM に寄せるかを意識します。

| 寄せるもの | 注意点 |
| --- | --- |
| プロパティ | `private set` やフィールドマッピングを検討する |
| コンストラクタ | ORM 用の空コンストラクタが必要になることがある |
| コレクション | 外部から自由に追加できない形にする |
| 遅延読み込み | 意図しない DB アクセスに注意する |

ORM と完全に切り離すほど実装は増えます。逆に、ORM に寄せすぎるとモデルの表現力が落ちます。

実務では、ドメインモデルの不変条件を守れる範囲で ORM に合わせることが多いです。

**ORM の都合より、ドメインモデルの不正状態を防ぐことを優先する**のが基準です。
