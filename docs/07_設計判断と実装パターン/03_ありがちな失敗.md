# ありがちな失敗

DDD の失敗は、構造を作りすぎることと、構造を作っただけで業務理解が進まないことの両方で起きます。

| 失敗 | 起きること |
| --- | --- |
| すべてを Entity にする | 値の制約が散らばる |
| Service に処理を集める | ドメインモデルが貧血になる |
| Aggregate を大きくする | 保存と変更の単位が重くなる |
| Repository をテーブル単位にする | Aggregate 境界が崩れる |
| レイヤーだけ先に作る | 業務用語がコードに現れない |

最初から完全なモデルを作ろうとしないことも大切です。会話、実装、テスト、運用を通じて、モデルは変わります。

**DDD は正しい構造を最初に当てる作業ではなく、モデルを育て続ける作業**です。
