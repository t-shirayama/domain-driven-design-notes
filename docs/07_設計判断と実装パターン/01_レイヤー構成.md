# レイヤー構成

DDD の実装では、責務ごとにレイヤーを分けることが多いです。代表的には Presentation、Application、Domain、Infrastructure に分けます。

| レイヤー | 主な責務 |
| --- | --- |
| Presentation | HTTP、UI、リクエストとレスポンス |
| Application | ユースケースの進行 |
| Domain | 業務ルール、モデル、不変条件 |
| Infrastructure | DB、外部 API、メッセージング |

重要なのはフォルダ名ではなく、変更理由を混ぜないことです。小さなアプリでは、物理プロジェクトを細かく分けずに名前空間やフォルダで分けても十分な場合があります。

**レイヤー分割は、業務ルールを技術詳細から守るために使う**ものです。
