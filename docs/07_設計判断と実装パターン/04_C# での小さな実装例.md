# C# での小さな実装例

小さな構成では、Domain に業務ルール、Application にユースケース、Infrastructure に Repository 実装を置きます。

```csharp
// Domain
public sealed class Order
{
    public OrderId Id { get; }
    public OrderStatus Status { get; private set; }

    public void Cancel()
    {
        if (Status == OrderStatus.Shipped)
            throw new InvalidOperationException("出荷済み注文はキャンセルできません。");

        Status = OrderStatus.Cancelled;
    }
}
```

```csharp
// Application
public sealed class CancelOrderUseCase(IOrderRepository orders)
{
    public async Task ExecuteAsync(OrderId orderId, CancellationToken ct)
    {
        var order = await orders.GetAsync(orderId, ct);
        order.Cancel();
        await orders.SaveAsync(order, ct);
    }
}
```

この例では、キャンセルできるかの判断を `CancelOrderUseCase` ではなく `Order.Cancel()` に置いています。

**ユースケースは流れを持ち、ドメインモデルは判断を持つ**ように分けます。
