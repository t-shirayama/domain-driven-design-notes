# ドメインの蒸留

ドメインの蒸留は、複雑な業務の中から本当に重要な部分を見つける活動です。全体を同じ濃さで理解しようとすると、Core Domain が埋もれます。

蒸留では、売上、差別化、変更頻度、業務担当者の関心、失敗時の影響を見ます。技術的に難しい場所が Core Domain とは限りません。

| 観点 | 見ること |
| --- | --- |
| 差別化 | 競合と違いを作るか |
| 変更頻度 | 事業上よく変わるか |
| 専門性 | 業務担当者の暗黙知が多いか |
| 影響 | 間違えると事業に大きく響くか |

蒸留した結果、Generic Subdomain は外部サービスへ任せる判断もあります。

**重要ではない場所を軽くすることで、重要な場所を深く設計できます。**
