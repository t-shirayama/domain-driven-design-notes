# 全部を DDD にしない

DDD は、複雑な業務ルールを扱うための設計方法です。すべての画面、すべての CRUD、すべてのバッチに同じ重さで適用すると、開発速度を落とすことがあります。

| 対象 | 方針 |
| --- | --- |
| 複雑な業務判断 | DDD を厚めに使う |
| 単純な参照画面 | Read Model や SQL 直読みを検討する |
| マスタ管理 | 薄い CRUD で十分な場合がある |
| 外部連携の変換 | 腐敗防止層を置く |

DDD を使わない判断も設計判断です。複雑さがない場所に複雑な構造を入れないようにします。

**DDD は全体に均等に塗るものではなく、複雑な部分へ集中的に使う**ものです。
