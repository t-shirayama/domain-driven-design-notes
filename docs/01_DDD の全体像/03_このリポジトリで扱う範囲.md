# このリポジトリで扱う範囲

このリポジトリでは、DDD の用語を実務で読み返しやすい粒度で整理します。理論の網羅よりも、設計判断で迷ったときに戻れるメモを目指します。

扱う範囲は、戦略的設計と戦術的設計の両方です。戦略的設計では、ユビキタス言語、境界づけられたコンテキスト、コンテキストマップを扱います。戦術的設計では、Entity、Value Object、Aggregate、Repository、Domain Event などを扱います。

C# のコード例は短くします。フレームワークの使い方を説明するためではなく、ドメインモデルや依存方向を思い出すために使います。

全体像は [draw.io 図](/domain-driven-design-notes/ddd-overview.drawio) にまとめます。本文中の小さな図は、差分管理しやすい Mermaid で補足します。

**このメモは、DDD を実装パターン集としてだけでなく、設計会話の進め方として整理します。**
