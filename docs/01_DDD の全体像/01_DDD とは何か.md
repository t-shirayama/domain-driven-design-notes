# DDD とは何か

DDD は Domain-Driven Design の略で、ソフトウェアの中心を技術ではなく業務の関心に置く設計方法です。画面、DB、フレームワークよりも先に、業務で扱う概念、言葉、ルール、境界を明らかにします。

DDD の目的は、複雑な業務をそのままコードに押し込むことではありません。業務を理解し、変更に耐えやすい単位へ分け、チームが同じ言葉で会話できる状態を作ることです。

| 観点 | DDD で見るもの |
| --- | --- |
| 言葉 | 業務で使う用語とコード上の名前を近づける |
| 境界 | 同じ言葉が同じ意味で使える範囲を分ける |
| ルール | 業務判断をドメインモデルに置く |
| 変更 | 変更理由が違うものを混ぜない |

**DDD は、業務知識をコードの中心に置くための設計活動**です。
